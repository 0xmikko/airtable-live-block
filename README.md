# AirTableLive
## Connect your web with AirTable data

<img width="1440" alt="1" src="https://storage.googleapis.com/airtable-live/intro.jpeg">

### Project repositories

https://github.com/MikaelLazarev/airtable-live-block - AirTable block code

https://github.com/MikaelLazarev/airtable-live-client - Front-end for landings

https://github.com/MikaelLazarev/airtable-live-server - Back-end

### Problem
AirTable users keeps their data like product catalog, clients sales in AirTable and has to hire developers to update 
their web resources if something changes.
  
### Solution
AirTableLive is nocode platform, which helps users to design their landings pages, connect product catalog, etc. and update them in one click!

<img width="1440" alt="1" src="https://storage.googleapis.com/airtable-live/story.jpeg">

### Customer story

John is entrepreneur, he owns a small pizzeria. John is AirTable fun, he keeps company data there, including menu, sales, etc.

With AirtableLive, John created a beautiful landing in a few minutes and connect it with company data.

John publishes seasonal menu updates, and other info with one click!

### How to use

**1.** Create landing structure for existing blocks.

<img width="1440" alt="1" src="https://storage.googleapis.com/airtable-live/actions/action-1.png">
Please, copy a schema table to have correct structure.     

**2.** Create / or connect your landing with AirTable data using Schema editor.

<img width="45%" alt="1" src="https://storage.googleapis.com/airtable-live/actions/action-2a.png">&nbsp;<img width="45%" alt="1" src="https://storage.googleapis.com/airtable-live/actions/action-2b.png">

**3.** Fill tables with actual data. You could check preview, which immediately renders current landing.

<img width="1440" alt="1" src="https://storage.googleapis.com/airtable-live/actions/action-3.png" />

**4.** Choose landing name and publish it on `https://airtable.live/<YOUR LANDING NAME>` by pressing `Publish` button

<img width="50%" alt="1" src="https://storage.googleapis.com/airtable-live/actions/action-4.png" />
